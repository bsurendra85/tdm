Real time binding between the form and the model, defaults to false.

When true, every value change event emitted from the from will trigger and update to the model.

When false (default) the form data model and the model instance are not bound, to update the model you need to invoke the TDMModelForm.commitToModel.

A hot bind is ONE WAY, from FORM UPDATES to the MODEL. Changes in the model will not reflect on the form.
