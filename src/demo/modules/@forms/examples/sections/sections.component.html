You can split the form multiple times and create sections using the slaveOf attribute.

A form must have a single master but can have any number of slaves. Slaves components does not emit all of the events fired by the dynamic form, those are fired from the master.

All slaves share the same TDMFormModel instance.

The following example shows 3 sections (3 dynamic-form component instances), all from the same form instance. The 2nd section is a container for 2 sections separated using a TAB component.
